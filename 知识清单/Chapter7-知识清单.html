7.1 无线与移动网络概述

无线与移动网络的背景 了解

无线（移动）电话用户数已远超固定（有线）电话用户；移动宽带连接设备数也已超过固定宽带连接设备数。蜂窝网络现已采用互联网协议栈，并引入 4G/5G、SDN 等技术。无线与移动网络面临两个重要且不同的挑战：无线（wireless）——通过无线链路进行通信；移动（mobile）——处理改变网络接入点的移动用户。

易错："无线"与"移动"是两个不同维度的概念，二者既可以独立存在也可以同时存在（见下文"无线网络分类"）。

无线网络的组成要素 重点

无线主机（wireless host）：笔记本电脑、智能手机、IoT 设备等，运行应用程序，可以是静止的（非移动）或移动的。无线并不总是意味着移动性——例如台式机通过无线网卡接入网络但从不移动。

基站（base station）：通常连接到有线网络，起中继作用，负责在其"蜂窝区域"中的有线网络和无线主机之间转发数据包，例如手机信号塔、802.11 接入点（AP）。主机与某个基站相关联（associate）。

无线链路（wireless link）：通常用于将移动设备连接到基站，也可用作骨干链路；由多路访问控制（MAC）协议协调链路接入；具有各种不同的传输速率、覆盖距离和使用频段。

典型无线网络标准对比 常考

按覆盖范围可大致分为：室内（10-30m，如 Bluetooth 2Mbps）、室外中距离（50-200m，如 802.11b/g/n）、室外中长距离（200m-4km，如 802.11ac/ax、4G LTE）、室外长距离（4km-15km，如 5G、802.11af/ah）。总体趋势是覆盖范围越大，可达到的最高数据速率通常越低。

无线网络的模式：基础结构与自组织 必考

模式特点
基础结构模式（infrastructure）基站将移动终端连接到有线网络；移动终端更换基站时发生切换（handoff），以维持有线网络连接
自组织模式（ad hoc）网络中无基站；节点只能向无线链路覆盖范围内的其他节点传输；节点之间自行组织成一个网络（自组网），并在节点间进行路由

无线网络分类（单跳/多跳 × 有/无基础设施） 重点

单跳多跳
有基础设施主机连接到基站（WiFi、蜂窝），基站连接到更大的互联网主机可能需通过多个无线节点中继才能连接到更大的互联网：网状网络（mesh network）
无基础设施没有基站，无法连接到更大的互联网（蓝牙、点对点网络）没有基站，也没有连接到更大的互联网；可能需要中继才能到达其他给定的无线节点：MANET、VANET

7.2 无线链路与网络特性

无线链路与有线链路的区别 必考

无线链路存在以下几个使通信变得更加"困难"的独特问题：

1）信号强度衰减（路径损耗）：无线电信号在介质中传播时逐渐衰减；

2）来自其他来源的干扰：许多设备（如 WiFi、蜂窝网络、电机）共享同一无线频率（如 2.4GHz），彼此产生干扰；

3）多路径传播：无线电信号从物体表面反射后，会在不同时刻到达同一目的地，造成信号失真。

信噪比与误码率（SNR / BER） 必考

信噪比（SNR）越大，越容易从噪声中提取出信号；误码率（BER）则是比特差错发生的概率。二者之间存在权衡（SNR vs BER）：

给定物理层技术：增大发送功率 --> SNR 增大 --> BER 降低；

给定 SNR：应选择满足 BER 要求前提下吞吐量最高的物理层调制方式（如 BPSK 1Mbps、QAM16 4Mbps、QAM256 8Mbps，SNR 越高可用的调制阶数越高）；

SNR 可能随移动性变化而变化，因此需要动态自适应物理层（调制技术、传输速率）以维持可接受的 BER。

隐藏终端问题与信号衰减问题 必考

设有 A、B、C 三个节点依次分布，B 位于 A 和 C 之间。

隐藏终端问题：B 与 A 彼此可以侦听到；B 与 C 彼此可以侦听到；但 A 与 C 由于障碍物遮挡（信号被完全阻断），彼此侦听不到——若 A、C 同时向 B 发送，会在 B 处发生冲突，而 A、C 双方都不知情。

信号衰减问题：B 与 A、B 与 C 彼此可以侦听到；但 A、C 因距离过远、信号强度衰减而彼此侦听不到，即 A 和 C 不知道他们的信号会在 B 处相互干扰。

易错：隐藏终端问题与信号衰减问题的现象相似（A、C 互相听不到导致 B 处冲突），但成因不同——前者是障碍物遮挡，后者是距离过远导致的信号强度衰减，二者都会促使 802.11 采用 CSMA/CA 而非 CSMA/CD。

所有 802.11 标准均使用 CSMA/CA 进行多址接入，并具有基础结构（基站）与自组网两种网络模式版本。当前主流为 Wi-Fi 6/6E；Wi-Fi 7（802.11be）已于近年开始商用部署。

802.11 LAN 体系结构：BSS 与 AP 重点

无线主机与基站（base station）通信，基站又称接入点（Access Point, AP）。AP 及与其关联的无线主机共同构成一个基本服务集（Basic Service Set, BSS），也称为一个蜂窝（cell）。基础结构网络模式下，BSS 包含无线主机与 AP，AP 连接到交换机或路由器进而接入 Internet；自组网（ad hoc）模式下，BSS 中只有主机，没有 AP。

信道选择与关联过程 重点

频谱被划分为不同频率的信道（channel），每个 AP 需选择一个信道工作；相邻的 AP 可能选择相同的信道，从而存在干扰的可能。主机必须与某个 AP关联（associate）才能通信，一般流程为：主机扫描信道，监听包含 服务集标识符（SSID）和 MAC 地址的信标帧（beacon frame）；选择一个 BSS 进行关联；可能需要进行身份认证；典型情形下随后运行 DHCP 获取 IP 地址等信息。

被动扫描与主动扫描 常考

方式过程
被动扫描（passive scanning）①各 AP 周期性发送信标帧（Beacon Frame）；②主机(H1)向选择的 AP 发送关联请求帧；③AP 向主机(H1)发送关联响应帧
主动扫描（active scanning）①主机(H1)主动广播探测请求帧（Probe Request Frame）；②各 AP 发送探测响应帧（Probe Response Frame）；③主机(H1)向选择的 AP 发送关联请求帧；④AP 向主机(H1)发送关联响应帧

CSMA/CA 协议原理 必考

目标：避免冲突（Collision Avoidance）而非像有线以太网 CSMA/CD 那样"边发送边检测冲突"。原因：①无线信道中发射信号强度远高于接收信号强度，难以在发送的同时侦听到冲突；②无法侦听到所有可能的冲突（隐藏终端、信号衰减问题）。

802.11 发送端流程：若监听到信道空闲达 DIFS（分布式帧间间隔）时间，则发送整个帧（无同时检测冲突）；若监听到信道忙，则开始随机退避计时，信道空闲时计时器倒计时，计时器超时后发送帧；若未收到 ACK，则增大随机退避区间并重复上述过程。

802.11 接收端：若正确接收到帧，则延迟 SIFS（短帧间间隔，SIFS < DIFS）时间后向发送端回送 ACK；由于隐藏终端问题的存在，ACK 确认机制是必要的。

RTS/CTS 冲突避免机制 必考

基本思想：允许发送端"预约（reserve）"信道而不是随机发送数据帧，从而避免长数据帧发生冲突。流程：①发送端先利用 CSMA 向 AP 发送一个很短的 RTS（request-to-send）帧——RTS 帧仍可能彼此冲突，但由于帧很短，代价很低；②AP 广播一个 CTS（clear-to-send）帧作为对 RTS 的响应，CTS 帧可被所有节点接收到，从而消除隐藏终端的影响，其他节点据此推迟发送；③发送端随后发送数据帧。利用很小的预约帧彻底避免了（长）数据帧的冲突。

易错：RTS 帧本身并非绝对不会冲突，只是因为帧很短，即使冲突代价也很小；真正被"保护"而避免冲突的是后续的数据帧。

802.11 MAC 帧结构与地址字段 重点

802.11 MAC 帧主要字段：帧控制（frame control）、持续时间（duration，用于 RTS/CTS 预约传输的时间）、地址 1~4、序号控制（seq control，用于处理可能的重传副本）、数据净荷（payload，0~2312 字节）、CRC 校验。帧控制字段包含协议版本、类型/子类型（区分 RTS、CTS、ACK、data 等帧类型）、ToDS、FromDS、More Frag、Retry、Power Mgt、More Data、WEP、Rsvd 等子字段。

移动性与高级功能 常考

同一子网内的移动：只要主机 H1 留在同一 IP 子网内，其 IP 地址可保持不变；交换机通过自学习机制——根据来自 H1 的帧，"记住"哪个交换机端口可到达 H1，从而正确转发帧。

速率适应（rate adaptation）：当移动终端远离基站导致 SNR 降低、BER 升高时，动态切换到更低阶的调制方式（如从 QAM256 切换到 QAM16 甚至 BPSK）以降低 BER，代价是速率随之降低。

功率管理（power management）：节点告知 AP"我要休眠，直到下一个信标帧"，AP 得知后不再向该节点发送帧；节点在下一个信标帧之前醒来；信标帧中包含 AP 目前缓存的、等待发送给各休眠移动终端的帧列表；若有待发送给该节点的帧，节点保持唤醒状态，否则再次进入休眠，直到下一个信标帧。

蓝牙（Bluetooth）个人区域网 了解

覆盖半径通常小于 10m，常用于替代线缆（鼠标、键盘、耳机），采用自组网（无基础设施）模式，使用 2.4-2.5GHz ISM 无线电频段，速率最高可达 4Mbps。设备角色分为主设备（master device）与客户设备（client device）：主设备轮询各客户设备，授予其传输许可；停放设备（parked device）处于非活跃（inactive）状态。

多路复用方式：TDM（625μs 时隙）+ FDM（发送方使用跳频扩频 FHSS：79 个频率信道，每个时隙使用一个信道，按伪随机顺序在信道间跳变；不在同一皮克网（piconet）内的设备仅可能干扰某些时隙）。停放模式：客户设备可"进入睡眠"（停放）并在稍后唤醒以节省电量；引导（bootstrapping）：节点即插即用地自组成一个皮克网。

易错点汇总

易错：无线 ≠ 移动。静止设备也可以通过无线方式接入网络（如台式机 WiFi 网卡）。
易错：802.11 的 CSMA/CA 不能像有线以太网 CSMA/CD 那样在发送的同时检测冲突，因为无线收发器发送信号强度远大于接收信号强度，且存在隐藏终端/信号衰减导致的"侦听盲区"。
易错：隐藏终端问题（障碍物遮挡）与信号衰减问题（距离过远）现象相似但成因不同。
易错：RTS 帧仍可能冲突，只是代价很小；数据帧才是被 RTS/CTS 机制真正保护、避免冲突的对象。
易错：802.11 数据帧有 4 个地址字段，具体取值取决于 ToDS/FromDS 两个比特的组合，而非固定不变。
