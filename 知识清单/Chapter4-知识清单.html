4.1 网络层概述

网络层的两大核心功能 必考

转发（forwarding）：路由器将分组从输入端口转移到合适的输出端口的动作，发生在数据平面，由硬件在纳秒级时间尺度完成，依据的是路由器本地的转发表。

路由选择（routing）：确定分组从源到目的所经过路径的过程，发生在控制平面，由运行在路由器（或集中控制器）上的软件在毫秒到秒级完成，路由算法的输出结果用于计算和更新转发表。

易错：转发和路由选择常被混淆。转发是"低层动作"（看一个分组、查表、送出），路由选择是"全局计算"（算出整个网络的路径）。转发表是路由选择算法的产物，但转发本身不需要再次计算路径。

数据平面与控制平面 重点

传统路由器中，路由选择算法本身也运行在路由器内部，控制平面是"每路由器一个"；而在软件定义网络（SDN）中，控制平面从路由器中抽离，由远程控制器统一计算并下发转发表，控制平面变为"逻辑集中"。

4.2 虚电路网络与数据报网络

虚电路网络（Virtual Circuit, VC） 必考

在数据传送前需要经过建立连接阶段，源到目的之间的路径在此阶段被确定，路径上每台路由器都要维护该连接的状态。每条虚电路由一个VCID（虚电路号）标识。

关键点：同一条虚电路在其经过的每段链路上使用的 VCID 可以不同，因为 VCID 只具有本地链路意义；分组每经过一台路由器，路由器都要根据自己的 VC 转发表把分组的 VCID 字段重写为下一段链路上对应的 VCID，然后从对应输出端口转发。

易错：很多同学误以为"同一条虚电路全程使用相同的 VCID"，这是错误的——VCID 是逐链路（link-by-link）分配和重写的，路由器的 VC 表中就存有"输入端口/输入VCID --> 输出端口/输出VCID"的映射。

数据报网络（Datagram Network） 重点

不需要建立连接，每个分组携带完整的目的地址，独立地根据转发表进行转发，同一条流的不同分组可能经过不同路径。IP 网络即采用此模式。转发表按目的地址前缀（最长前缀匹配）组织，而不是按连接组织。

VC 与数据报网络对比 高频

特性虚电路网络数据报网络
建立连接需要不需要
路由器状态每条连接维护状态无连接状态
分组头部字段VCID目的地址
路径固定可能变化
典型代表ATM、帧中继Internet（IP）

4.3 路由器体系结构

路由器四大组成部分 必考

输入端口（物理层/链路层处理、查找转发表、决定输出端口）、交换结构（将分组从输入端口连接到输出端口）、输出端口（缓存、调度、发送）、路由选择处理器（运行路由协议、维护路由表、计算转发表）。

队头阻塞（HOL Blocking） 必考

当输入端口采用FIFO 队列，且交换结构速率有限（一次只能处理一个分组）时，排在队首的分组若因目的输出端口繁忙而无法转发，会阻塞其后面排队的、原本可以转发到其他空闲输出端口的分组，造成不必要的等待和丢包，即使这些后面的分组本身没有拥塞。

易错：HOL 阻塞是"输入端口排队"带来的问题，与"输出端口拥塞"不同；解决办法之一是采用虚拟输出队列（VOQ），为每个输出端口在输入端口维护单独的队列。

三种交换结构 重点

1）经内存交换：分组在 CPU 控制下从输入端口拷贝到内存，再从内存拷贝到输出端口，是第一代路由器的方式，交换速率受限于内存带宽（需两次穿越总线）。

2）经总线交换：输入端口通过共享总线直接将分组送到输出端口，不经过路由选择处理器，但同一时刻总线只能传送一个分组，交换带宽受总线带宽限制。

3）经互联网络交换：使用如纵横式（crossbar）等多路径网络，允许多个分组并行地从不同输入端口传送到不同输出端口，克服了总线带宽的限制，是高性能路由器的主流方案。

输出端口调度规则 了解

常见调度规则：FIFO（先到先服务）、优先级排队（按类别分高低优先级队列）、循环（RR）（各类别轮流服务）、加权公平排队（WFQ）（按权重共享带宽，兼顾公平性与差异化服务）。当输出端口发送速率小于到达速率之和时，需要缓存，缓存耗尽则丢包。

4.4 网际协议 IP

IPv4 数据报格式 必考

固定首部 20 字节，主要字段：版本（4）、首部长度（以 4 字节为单位，无选项时为 5，即 20 字节）、区分服务（TOS/DiffServ）（服务质量标记，一般置 0）、总长度（首部+数据，字节，最大 65535）、标识（Identification）、标志（DF 不分片位、MF 更多分片位）、片偏移、生存时间 TTL、协议（指出上层协议，如 TCP=6，UDP=17）、首部校验和、源 IP 地址、目的 IP 地址、可选的选项字段与填充。

TTL 字段 重点

TTL 用于防止分组在网络中无限循环转发：每经过一台路由器，TTL 减 1，减到 0 时该路由器丢弃分组并可向源发送 ICMP 超时报文。由于 TTL 每跳都变化，首部校验和必须逐跳重新计算。

易错：TTL 不是"生存时间的秒数"，而是"最大可经过的路由器跳数"，名字容易让人误解为计时器。

IP 分片与重组 必考

当数据报长度超过链路的最大传输单元（MTU）时，路由器需要将其分片。设数据报总长度为 $L$（字节，含 20 字节首部，假设无选项），链路 MTU 为 $M$，则：

每片最大数据部分长度：$$d = \left\lfloor \frac{M - 20}{8} \right\rfloor \times 8$$

分片数：$$n = \left\lceil \frac{L - 20}{d} \right\rceil$$

第 $i$ 片（$i = 1, 2, \dots, n$）的片偏移（以 8 字节为单位计）：$$F_i = \frac{d}{8} \times (i - 1)$$

第 $i$ 片的总长度：$$L_i = \begin{cases} d + 20, & i < n \\ L - (n-1)d, & i = n \end{cases}$$

更多分片标志：$$MF_i = \begin{cases} 1, & i < n \\ 0, & i = n \end{cases}$$

所有分片共享同一个标识（Identification）字段值，目的主机据此将属于同一原始数据报的分片重新组装。只有最后一片的 MF 为 0。

易错：片偏移单位是 8 字节而不是 1 字节，计算时容易漏掉除以 8 或乘以 8 这一步；同时片长度计算要注意"数据部分"和"含首部总长度"的区别。

IPv4 编址：分类编址 重点

类别首字节范围网络号位数主机号位数
A类1–126824
B类128–1911616
C类192–223248
D类224–239组播地址
E类240–255保留（实验用）

子网划分与子网掩码 必考

子网掩码由连续的 1（网络位+子网位）和连续的 0（主机位）组成。将 IP 地址 与子网掩码逐位"与"（AND）运算，即得到该地址所在的子网网络地址。

例如 IP 地址 140.20.34.100，掩码 255.255.240.0（即 /20），二进制 34 = 00100010，与掩码高 4 位 1111 相与后得到子网部分为 00100000 = 32，因此子网地址为 140.20.32.0。

CIDR 与最长前缀匹配 必考

无类别域间路由（CIDR）采用 a.b.c.d/x 记法，$x$ 为网络前缀位数，打破了 A/B/C 类的固定边界，支持路由聚合（超网）：多个具有相同前缀的小网络可以聚合为一条更短前缀的路由，减少路由表条目。

转发时若一个目的地址同时匹配多条路由（前缀长度不同），路由器必须选择前缀最长的那条，即最长前缀匹配（Longest Prefix Match）原则。

易错：最长前缀匹配容易被误记为"选择第一条匹配的路由"，实际上必须比较所有匹配路由的前缀长度，取最长者，与路由表中条目的先后顺序无关。

特殊 IP 地址 高频

4.5 IPv6

IPv6 产生的动机 重点

1）IPv4 地址空间即将耗尽（32 位，约 43 亿个地址）；2）简化首部以加快路由器处理速度；3）更好地支持服务质量（QoS）。

IPv6 首部特点 必考

固定40 字节首部（不含选项，采用扩展首部替代 IPv4 的可变选项），地址长度扩展为128 位。相比 IPv4，IPv6 取消了首部校验和字段（由链路层和传输层保证差错检测，减少每跳处理开销），并且不再支持路径中间路由器分片（分片只能由源主机完成，若中间链路 MTU 不足则丢弃并返回 ICMPv6 差错）。

IPv6 地址表示 了解

128 位地址通常写成 8 组 4 位十六进制数（冒号分十进制），如 2001:0db8:0000:0000:0000:ff00:0042:8329；可压缩表示——每组前导 0 可省略，连续多组全 0 可用 :: 缩写一次；IPv4 兼容地址可写作 ::ffff:192.0.2.1。

IPv6 地址类型 了解

单播（一对一）、组播（一对多，替代 IPv4 广播）、任播（一对多中最近一个，anycast）。IPv6 中取消了广播地址类型，广播功能由组播替代。

IPv4 到 IPv6 过渡：隧道技术 重点

在 IPv4 网络尚未全部升级的过渡期，可采用隧道（tunneling）技术：在 IPv6 分组穿越纯 IPv4 网络区域时，将整个 IPv6 数据报封装进 IPv4 数据报的数据部分传输，到达隧道另一端的 IPv6 路由器后再解封装还原。

4.6 IP 相关协议

地址解析协议 ARP 必考

ARP 用于在同一局域网内将已知的 IP 地址解析为 MAC 地址。主机以广播帧（目的 MAC 全为 FF-FF-FF-FF-FF-FF）发送 ARP 查询，局域网内所有主机都会收到，只有 IP 地址匹配的目的主机以单播方式回复其 MAC 地址。查询结果缓存在本机的 ARP 表中，每条记录有一个生存时间（约 20 分钟），超时后需重新查询。

跨局域网通信中的地址变化 高频

数据报从源主机经过多台路由器到达目的主机的过程中，源 IP 地址和目的 IP 地址始终保持不变，但源 MAC 地址和目的 MAC 地址在每一跳都会被重写为当前链路两端设备的 MAC 地址（依赖每一跳的 ARP 解析）。

易错：初学者常误以为 MAC 地址也是端到端不变的，实际上只有 IP 地址端到端不变，MAC 地址是"逐跳"变化的，这是理解路由器转发过程的关键。

网际控制报文协议 ICMP 必考

ICMP 用于报告网络层的差错和提供网络诊断信息，分为两大类：

差错报告报文（5 种）：目的不可达、源点抑制（拥塞时降速）、超时（TTL 减为 0 或分片重组超时）、参数问题（首部字段有误）、改变路由（重定向）。

询问报文（2 对）：回送请求/回答（ping 使用）、时间戳请求/回答。

不发送 ICMP 差错报文的例外情况：1）对 ICMP 差错报文本身不再发送 ICMP 差错报文（防止无限循环）；2）对非第一个分片的数据报不发送；3）对具有组播地址的数据报不发送；4）对具有特殊地址（如 127.0.0.0 或 0.0.0.0）的数据报不发送。

易错：这四条例外情况是高频考点，容易漏记"非第一个分片"和"组播地址"这两条。

traceroute 工作原理 重点

源主机依次发送 TTL = 1, 2, 3, ... 的探测分组：TTL 耗尽的路由器返回 ICMP 超时报文，从而依次探测出路径上每一跳的地址；当分组到达目的主机时，由于目的端口通常未被使用，目的主机返回 ICMP 端口不可达报文，源主机据此判断已经到达终点，探测结束。

动态主机配置协议 DHCP 必考

DHCP 让主机即插即用地自动获取 IP 地址等配置信息，采用应用层的 route-d 守护进程实现，基于UDP封装，通过广播方式完成四条报文交互：

1）DHCP discover（主机广播发现服务器）--> 2）DHCP offer（服务器广播提供的配置）--> 3）DHCP request（主机广播确认选择某个提供）--> 4）DHCP ACK（服务器广播确认租约）。

网络地址转换 NAT 必考

NAT 使得整个内部网络对外只使用一个（或少量）公网 IP 地址，通过维护一张 NAT 转换表（记录内部 IP:端口 与 外部 IP:端口 的映射）实现端口级别的地址转换。

动机：1）缓解公网 IP 地址短缺；2）内部可以任意更改主机 IP 编址而不需通知外部（编址灵活性）；3）隐藏内部网络结构，一定程度提升安全性。

由于端口号字段为 16 位，理论上一个公网 IP 最多可支持约 6 万多（2^16 约 65536）个并发端口映射连接。

争议：1）违反了网络端到端原则；2）路由器本应只处理到第三层（网络层），NAT 却修改了传输层端口号，违反分层设计；3）破坏了 P2P 应用直接建立连接的能力（内网主机不可直接被外部发起连接）；4）反对者认为 IPv6 地址空间充足即可从根本上解决地址短缺问题，无需 NAT。

NAT 穿越（NAT Traversal）解决方案：1）静态配置端口转发（port forwarding），在 NAT 路由器上手动配置外部端口到内部主机端口的固定映射；2）UPnP（IGD，Internet Gateway Device）协议，允许内网主机自动向 NAT 路由器请求分配并配置端口映射；3）借助中继（Relay）服务器转发数据（如 Skype 早期采用的方式），双方都主动连接到公网上的第三方中继服务器进行中转通信。

4.7 路由选择算法

路由选择问题的抽象 重点

网络被抽象为图 $G = (N, E)$，$N$ 为路由器（节点）集合，$E$ 为链路（边，带权重代表代价）集合。路由算法的目标是找出源到目的代价最小的路径。

按信息获取方式分为：全局路由选择算法（每个节点掌握全网拓扑和链路代价，如链路状态算法）与分散式路由选择算法（节点只知道物理相连邻居的信息，通过与邻居反复交换信息迭代计算，如距离向量算法）。按是否随流量/拓扑变化调整分为静态路由选择与动态路由选择。

链路状态路由选择算法（LS，Dijkstra） 必考

每个节点通过洪泛方式获得全网拓扑和所有链路代价，然后独立运行 Dijkstra 最短路径算法，以自身为源节点计算到所有其他节点的最短路径。

算法维护 $D(v)$（当前从源到 $v$ 的最短路径估计代价）和 $p(v)$（$v$ 的前驱节点），每轮从未确定集合中选取 $D(v)$ 最小的节点加入已确定集合 $N'$，并用其更新邻居的 $D$ 值，直至所有节点都被确定。朴素实现下，共需 $n$ 轮迭代，每轮遍历所有节点找最小值，因此时间复杂度为 $$O(n^2)$$（使用堆等数据结构优化可降至 $O(n\log n)$）。

距离向量路由选择算法（DV，Bellman-Ford） 必考

基于 Bellman-Ford 方程：$$d_x(y) = \min_{v} \{ c(x,v) + d_v(y) \}$$ 其中 $v$ 遍历 $x$ 的所有邻居。每个节点只与直接相连的邻居周期性或触发式交换自己的距离向量，收到邻居更新后按方程重新计算，若结果变化则继续向自己的邻居传播，直至全网收敛。

无穷计数问题（Count-to-Infinity） 必考

当某条链路的代价增大或失效时，DV 算法可能出现坏消息传播极慢的问题：两个节点互相以对方（此时已失效）的路径作为参照不断微小递增各自的距离估计，代价值要经过很多轮迭代才能"数"到真正的（很大的或无穷大的）值，这种现象称为无穷计数问题。

毒性逆转（poisoned reverse）：若 $Z$ 通过 $Y$ 到达 $X$，则 $Z$ 向 $Y$ 通告的到 $X$ 的距离设为无穷大，从而防止 $Y$ 用经过 $Z$ 的路径反过来"欺骗"自己。但毒性逆转只能解决两节点之间形成的环路，对于涉及三个及以上节点的更大环路仍然无法根治。

RIP 协议中采用最大跳数限制缓解该问题：将 16 定义为"无穷大"（不可达），从而限制无穷计数持续的轮数上限。

易错："毒性逆转彻底解决了无穷计数问题"是错误说法，它只能应对二节点环路，多节点环路仍可能发生无穷计数，因此距离向量协议通常还需配合最大跳数等其他机制。

4.8 层次路由与路由协议

自治系统与层次路由 必考

因特网规模巨大，若将其视为单一网络运行统一路由算法，将带来规模问题（路由表和消息量爆炸）和管理自治性问题（各网络管理者希望自行控制内部路由策略，不愿受制于统一算法）。因此因特网被划分为许多自治系统（AS，Autonomous System），每个 AS 内部运行自己选择的域内路由选择协议（intra-AS）；AS 之间的连接由网关路由器负责，运行域间路由选择协议（inter-AS），两者共同决定路由器最终的转发表。

热土豆路由（Hot-Potato Routing） 重点

当一个 AS 内部到达外部目的网络有多个出口网关可选时，路由器选择自身到该网关之间域内路径代价最小的那个网关进行转发，尽快把分组"甩出"本 AS，而不关心分组离开本 AS 之后的路径长短。

RIP（路由信息协议） 重点

基于距离向量算法，度量为跳数（hop count），最大有效跳数为 15，16 表示不可达（无穷大）。每隔约 30 秒通过 UDP 广播一次路由更新（advertisement）报文，每个报文最多包含 25 个目的网络的信息。若 180 秒未收到某邻居的更新，则认为该邻居不可达/链路失效。RIP 由应用层的 route-d 守护进程实现。

OSPF（开放最短路径优先） 必考

基于链路状态算法，各路由器泛洪链路状态信息，本地运行 Dijkstra 算法计算最短路径。相比 RIP 的优势：1）支持安全认证（防止恶意路由通告）；2）支持到同一目的地的多条等代价路径并可负载均衡；3）为不同类型的业务提供不同 TOS 值对应不同度量的支持；4）集成对组播路由（MOSPF）的支持。

大型 AS 内部可采用层次化 OSPF（RFC 2328）：划分为多个区域（Area），其中一个必须是主干区域（Backbone Area）；连接主干与其他区域的路由器称为区域边界路由器（ABR）；负责本区域内部路由的称为主干路由器（BR）；连接到其他 AS 的称为自治系统边界路由器（ASBR）。

BGP（边界网关协议） 必考

BGP 是因特网事实上的域间路由选择协议标准，分为两种会话：eBGP（运行在不同 AS 的网关路由器之间，用于学习到达外部网络的可达性）与 iBGP（运行在同一 AS 内部路由器之间，用于将 eBGP 学到的外部路由信息扩散到本 AS 内所有路由器）。

BGP 路由通告中携带重要的路径属性：AS-PATH（记录该路由所经过的 AS 序列，也用于检测和避免环路）与 NEXT-HOP（进入被通告网络的下一跳路由器接口地址）。

当存在多条通往同一目的网络的路由时，BGP 按以下顺序选择最优路由：1）本地偏好值（local preference）最高者优先（体现策略偏好）；2）若本地偏好相同，选择 AS-PATH 最短者；3）若仍相同，选择 NEXT-HOP 路由器（热土豆路由）距离最近者；4）若仍无法区分，则依据一组附加准则（如 BGP 标识符大小等）确定。

BGP 本质上是策略路由（policy-based routing）：AS 之间存在商业关系（如提供商-客户、对等关系），一个 AS 通常不会将从一个提供商学到的路由转发给另一个提供商（除非本身是转接 AS），因此路由的选择和是否通告都受到商业策略的过滤，而不仅仅是"最短路径"。

易错点汇总

  • 虚电路的 VCID 是逐链路重写的，不是全程不变的。
  • A 类私有地址是 10.x.x.x，B 类私有地址才是 172.16–172.31.x.x。
  • 片偏移单位是 8 字节，不是 1 字节。
  • 跨网络通信中 IP 地址端到端不变，MAC 地址逐跳改变。
  • 毒性逆转只能解决二节点环路，不能根治无穷计数问题。
  • 最长前缀匹配与路由表条目顺序无关，只看前缀长度。
  • ICMP 差错报文有四种不发送的例外情形（对 ICMP 差错本身、非第一分片、组播地址、特殊地址）。

本章总结

网络层的核心是在转发（数据平面）与路由选择（控制平面）之间建立联系：转发依赖路由器体系结构与转发表，路由选择依赖 LS/DV 等分布式算法计算出最短路径。IP 协议族（IPv4/IPv6、ARP、ICMP、DHCP、NAT）解决了寻址、地址解析、差错报告、自动配置与地址复用等具体问题；而因特网规模巨大催生了以自治系统为单位的层次化路由体系，域内（RIP/OSPF）与域间（BGP）路由协议分工协作，共同构成了全球因特网的路由基础设施。
